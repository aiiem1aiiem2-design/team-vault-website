최종 점검을 수행한다.

실행 절차:
1. 현재 브랜치를 확인한다.
2. 변경 파일 목록을 확인한다.
3. lint를 실행한다.
4. test를 실행한다.
5. build를 실행한다.
6. 핵심 UI 흐름을 점검한다.
7. 기존 기능 영향 여부를 점검한다.
8. 사용자 검토용 보고서를 작성한다.
9. 승인 전까지 commit, push, merge를 하지 않는다.

반드시 지킬 것:
- 사용자 승인 전 git 작업 금지
- 검증 명령이 없으면 없다고 명확히 보고
- 실패한 검증은 숨기지 말고 그대로 보고

결과 형식:
1. 현재 브랜치
2. 작업 요약
3. 변경 파일
4. lint 결과
5. test 결과
6. build 결과
7. UI 점검 결과
8. 기존 기능 영향
9. 남은 문제
10. 승인 요청
