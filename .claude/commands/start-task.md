새 웹사이트 작업을 시작한다.

실행 절차:
1. 현재 브랜치를 확인한다.
2. main, master, production, prod 브랜치면 작업용 브랜치를 생성하거나 생성 제안한다.
3. 요청을 짧게 요약한다.
4. 작업 범위와 제외 범위를 정리한다.
5. 관련 파일을 파악한다.
6. 구현 전에 간단한 작업 계획을 제시한다.
7. 그다음 작업을 시작한다.

반드시 지킬 것:
- 메인 브랜치에서 직접 작업하지 않는다.
- 요청 범위를 먼저 정리한다.
- 기존 기능 영향 가능성을 먼저 생각한다.

결과 형식:
1. 현재 브랜치
2. 요청 요약
3. 작업 범위
4. 제외 범위
5. 수정 예상 파일
6. 시작 계획
