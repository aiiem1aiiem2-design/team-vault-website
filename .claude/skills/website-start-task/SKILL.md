[목표]
웹사이트 작업을 시작하기 전에 브랜치, 범위, 관련 파일을 정리한다.

[반드시 할 일]
- 현재 브랜치를 확인한다.
- main, master, production, prod 브랜치인지 확인한다.
- 메인 브랜치라면 작업 브랜치를 생성하거나 생성 제안한다.
- 요청을 짧게 요약한다.
- 작업 범위와 제외 범위를 정리한다.
- 수정 예상 파일을 정리한다.
- 시작 전 계획을 제시한다.

[금지]
- 메인 브랜치에서 직접 작업 시작 금지
- 범위 정리 없이 바로 구현 시작 금지

[결과 형식]
1. 현재 브랜치
2. 요청 요약
3. 작업 범위
4. 제외 범위
5. 수정 예상 파일
6. 시작 계획
