[목표]
웹사이트 작업 완료 후 최종 점검하고 사용자에게 먼저 보고한다.

[반드시 할 일]
- 현재 브랜치를 확인한다.
- 변경 파일 목록을 확인한다.
- lint를 실행한다.
- test를 실행한다.
- build를 실행한다.
- 핵심 UI 흐름을 점검한다.
- 기존 기능 영향 여부를 점검한다.
- 결과를 사용자에게 먼저 보고한다.
- 사용자 승인 전 commit, push, merge를 하지 않는다.

[검증 원칙]
- 검증이 성공했는지 실패했는지 명확히 쓴다.
- 검증 명령이 없으면 없다고 쓴다.
- 실패한 항목은 숨기지 않는다.

[결과 형식]
1. 현재 브랜치
2. 작업 요약
3. 변경 파일
4. lint 결과
5. test 결과
6. build 결과
7. UI 점검 결과
8. 기존 기능 영향
9. 남은 문제
10. 승인 요청
