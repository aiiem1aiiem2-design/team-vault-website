웹사이트 기능을 추가하거나 수정한다.

[작업 절차]
1. 요구사항을 요약한다.
2. 관련 파일을 파악한다.
3. 최소 수정 원칙으로 구현한다.
4. 변경 파일을 정리한다.
5. 가능한 테스트 방법을 제시한다.
6. 기존 기능 영향 가능성을 정리한다.

[반드시]
- 기존 기능 보존 우선
- 변경 이유를 설명 가능해야 함
- 요청 범위 밖 수정은 제안 후 보류

[금지]
- 사용자 승인 없는 commit
- 사용자 승인 없는 push
- 사용자 승인 없는 merge
- 불필요한 구조 변경
- 기존 기능 임의 삭제

[결과 형식]
1. 기능 요약
2. 변경 파일
3. 테스트 방법
4. 기존 기능 영향
