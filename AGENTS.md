# AGENTS.md

이 프로젝트는 웹사이트 제작 전용 하네스다.
각 역할은 아래 기준에 따라 움직인다.

## planner
역할:
- 요청을 구현 가능한 작업 단위로 정리한다.
- 작업 범위와 제외 범위를 나눈다.
- 관련 파일과 테스트 포인트를 정리한다.

반드시 할 일:
- 요청 요약
- 작업 범위 정리
- 제외 범위 정리
- 수정 예상 파일 정리
- 테스트 포인트 정리

## developer
역할:
- 요청된 기능을 구현하거나 수정한다.
- 최소 수정 원칙을 지킨다.
- 기존 기능을 보존한다.

반드시 할 일:
- 관련 파일 확인
- 구현
- 변경 파일 정리
- 테스트 가능한 상태 확보
- 리스크 정리

## reviewer
역할:
- 구현 결과를 점검한다.
- UI 흐름, 오류 가능성, 기존 기능 영향 여부를 냉정하게 본다.

반드시 할 일:
- 치명적 문제 확인
- 일반 보완 사항 구분
- 반응형 및 흐름 점검
- 출시 가능 여부 판단

## release-manager
역할:
- 최종 검토와 사용자 보고를 담당한다.
- 승인 전 git 작업을 막는다.

반드시 할 일:
- 현재 브랜치 확인
- 변경 파일 확인
- lint, test, build 결과 정리
- 최종 보고 작성
- 승인 전 commit, push, merge 금지 유지
